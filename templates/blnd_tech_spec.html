Technical Architecture

Overview

Bland.ai is a cutting-edge Voice AI platform that enables developers to create sophisticated voice applications. This project leverages Bland.ai to enable users to manage to-do tasks, reminders, and calendar events through natural voice interactions.

Users can create, update, and delete their tasks, reminders, and events simply by speaking. All data is stored in a PostgreSQL database and seamlessly integrated with the user's Google Calendar and Gmail via the Google Calendar API, ensuring real-time synchronization across platforms. The system also provides a web interface for manual management alongside voice interactions.

API Routes (routes.py)

from flask import Blueprint, request, jsonify
from shared.helpers import get_validated_tool_call
from .models import BlndTodo, BlndReminder
from shared.schemas import TodoResponse, ReminderResponse
from shared.google_calendar import get_calendar_service

Blnd_todo_bp = Blueprint('blnd_todo', __name__, url_prefix='/blnd_todo')

@Blnd_todo_bp.route('/create_todo', methods=['POST'])
def create_todo():
    tool_call = get_validated_tool_call('createTodo')
    args = tool_call.function.arguments
    title = args.get('title', '')
    description = args.get('description', '')
    
    # Create todo in database
    todo = BlndTodo(title=title, description=description)
    db.session.add(todo)
    db.session.commit()
    db.session.refresh(todo)
    
    # Sync with Google Calendar
    try:
        calendar_service = get_calendar_service()
        google_event_id = calendar_service.create_event(
            title=f"TODO: {title}",
            description=description or "Task from Bland.ai Todo System"
        )
        if google_event_id:
            todo.google_calendar_event_id = google_event_id
            db.session.commit()
    except Exception as e:
        print(f"Failed to sync with Google Calendar: {e}")
    
    return jsonify({'results': [{'toolCallId': tool_call.id, 'result': 'success'}]})

Helpers (shared/helpers.py)
from flask import request, abort
from pydantic import ValidationError as PydanticValidationError
from .schemas import ToolRequest, ToolCall

def get_validated_tool_call(expected_function_name: str) -> ToolCall:
    """
    Validates the incoming JSON request against the ToolRequest schema
    and returns the specific tool call matching the expected function name.
    """
    json_data = request.get_json()
    if not json_data:
        abort(400, description="Invalid JSON payload.")

    try:
        # Validate the entire request structure
        tool_req = ToolRequest(**json_data)
    except PydanticValidationError as e:
        abort(400, description=f"Invalid request format: {e.errors()}")

    # Find the specific tool call we are looking for
    for tool_call in tool_req.message.toolCalls:
        if tool_call.function.name == expected_function_name:
            # If arguments are a string, parse them into a dict
            if isinstance(tool_call.function.arguments, str):
                tool_call.function.arguments = request.json_loads(tool_call.function.arguments)
            return tool_call

    abort(400, description=f"Tool call with function name '{expected_function_name}' not found.")
Schemas (shared/schemas.py)
import datetime as dt
from typing import Union, Dict, Any
from pydantic import BaseModel, ValidationError as PydanticValidationError

# --- Common Schemas for Tool Calling ---

class ToolCallFunction(BaseModel):
    name: str
    arguments: Union[str, Dict[str, Any]]

class ToolCall(BaseModel):
    id: str
    function: ToolCallFunction

class Message(BaseModel):
    toolCalls: list[ToolCall]

class ToolRequest(BaseModel):
    """A generic request model for any tool-calling service."""
    message: Message

# --- Common Response Schemas ---

class TodoResponse(BaseModel):
    id: int
    title: str
    description: Union[str, None]
    completed: bool
    model_config = {'from_attributes': True}

Bland.ai Integration Guide

Bland.ai Configuration & Deployment

1. Bland.ai Platform Setup

Start by creating an account on the Bland.ai platform and logging in to access the development dashboard.

2. Create Tools

Once logged in, navigate to the Tools section and create the necessary tools for todo management.

3. Tool Functions

Todo Functions

  • • getTodos
  • • completeTodo
  • • deleteTodo
Reminder Functions
  • • addReminder
  • • getReminders
  • • deleteReminder
Calendar Functions
  • • addCalendarEntry
  • • getCalendarEntries
  • • deleteCalendarEntry

4. Create Conversational Pathways

Create Pathway Nodes
  • • Create nodes for each conversational step (e.g., greeting, ask for todo details, confirm creation)
  • • Define node actions and expected user responses
Connect Nodes by Call Flow
  • • Link nodes according to the desired conversation flow
  • • Use conditions and transitions to handle user intents (e.g., create, list, complete, delete todos)
  • • Ensure each pathway covers all tool calls and fallback scenarios

5. Deploy

The deployment process follows a standard path from development to production:

  • Development Pathway: Start with development versions for the pathway to build and test new features.
  • Staging Pathway: Once functionality is verified, deploy the pathway to a staging environment for further testing and validation.
  • Production Pathway: After successful staging, promote the pathway to the production environment to make it live for all users.
